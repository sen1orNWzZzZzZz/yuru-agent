# 智能旅游多Agent规划系统 V2 升级说明

> 版本: 2.1.0 | 日期: 2026-05-08 | Python版

---

## 一、V2 核心升级概览

V2版本在V1基础上进行了大规模架构升级，核心目标是**接入真实LLM进行智能推理**，同时保持**Mock数据的完整性和可用性**，实现**LLM/Mock双模式自动切换**。

### 主要新增功能

| 序号 | 功能 | 说明 |
|------|------|------|
| 1 | **LLM集成** | 支持OpenAI格式API，兼容OpenAI/SiliconFlow/Azure/自托管vLLM |
| 2 | **双模式切换** | LLM智能推荐 / Mock数据回退，自动降级，无需人工干预 |
| 3 | **Agent抽象基类** | BaseAgent统一接口，新增Agent只需实现2个方法 |
| 4 | **管理后台** | `/admin`页面，支持LLM配置CRUD + 实时连接测试 |
| 5 | **Mock数据扩充** | 15个城市（195酒店 + 120餐厅 + 89景点 + 21小红书笔记） |
| 6 | **反水军样本** | 8条水军笔记（刷量/营销号/模板/僵尸粉） |
| 7 | **架构可扩展性** | 新增Agent只需继承BaseAgent + 注册Prompt模板 |
| 8 | **API Key安全** | 回显脱敏（前8位+****+后4位） |

### 2.2 Agent双模式工作原理

```
用户请求
  |
  v
AgentExecutor.run(prompt)
  |
  v
Step 1: CoordinatorAgentV2.execute(prompt)
  |-- 检测LLM配置存在？
  |   ├── Yes → 调用LLM生成需求解析
  |   └── No  → Mock正则提取
  |
  v
Step 2: ThreadPool并行执行4个推荐Agent
  |-- HotelAgentV2.execute()
  |-- RestaurantAgentV2.execute()
  |-- AttractionAgentV2.execute()
  |-- TransportAgentV2.execute()
  |   每个Agent内部:
  |   if use_llm:
  |       1. _build_params() 构建参数
  |       2. AgentPrompts.get_prompt() 获取模板
  |       3. LLMClient.chat_with_retry() 调用LLM
  |       4. LLM生成reasoning + Mock数据兜底
  |   else:
  |       _mock_fallback() 直接返回Mock数据
  |
  v
Step 3: RiskAgentV2.execute() 风控检测
  |
  v
返回完整规划结果
```

## 六、LLM配置指南

### 6.1 支持的API服务商

| 服务商 | Base URL | 推荐模型 |
|--------|----------|----------|
| OpenAI官方 | `https://api.openai.com/v1` | gpt-4o-mini / gpt-4o |
| SiliconFlow | `https://api.siliconflow.cn/v1` | Qwen/Qwen2.5-72B-Instruct |
| Azure OpenAI | `https://{实例名}.openai.azure.com/openai/deployments/{部署名}` | 部署时指定 |
| 自托管(vLLM) | `http://localhost:8000/v1` | 启动时指定 |

### 6.2 配置步骤

1. 打开管理后台 `http://localhost:8000/admin`
2. 点击"+ 新增配置"
3. 填写API Key、Base URL、模型名称
4. 点击"🔌 测试连接"验证
5. 点击"💾 保存配置"
6. 勾选"启用LLM"开关

### 6.3 降级策略

- **无配置时**：所有Agent自动使用Mock模式
- **连接失败时**：Agent自动回退到Mock模式
- **手动关闭**：取消"启用LLM"勾选，强制Mock模式
- **LLM返回异常JSON**：解析失败时回退Mock模式

## 七、扩展开发指南

### 7.1 如何新增一个Agent（以ShoppingAgent为例）

**Step 1: 创建Agent类**

```python
# app/agents/shopping_agent_v2.py
from app.agents.base import BaseAgent

class ShoppingAgentV2(BaseAgent):
    agent_type = "shopping"
    agent_name = "购物推荐Agent"

    def _build_params(self, **kwargs) -> Dict[str, Any]:
        return {
            "destination": kwargs.get("destination", "杭州"),
            "budget": kwargs.get("budget") or "未指定",
        }

    def _mock_fallback(self, **kwargs) -> Dict[str, Any]:
        # 实现Mock数据回退
        return {
            "recommendations": [...],
            "reasoning": "...",
            "total_found": 0,
        }
```

**Step 2: 注册Prompt模板**

```python
# app/llm/prompts.py
class AgentPrompts:
    SHOPPING_SYSTEM = "你是一个购物推荐专家..."
    SHOPPING_USER_TEMPLATE = "请推荐{destination}的购物场所..."

    @classmethod
    def get_prompt(cls, agent_type, user_params):
        prompt_map = {
            # ... 原有映射 ...
            "shopping": (cls.SHOPPING_SYSTEM, cls.SHOPPING_USER_TEMPLATE),
        }
        # ...
```

**Step 3: 注册到执行器**

```python
# app/agents/base.py
class AgentExecutor:
    def _create_agents(self):
        from app.agents.shopping_agent_v2 import ShoppingAgentV2
        return {
            # ... 原有Agent ...
            "shopping": ShoppingAgentV2(self.llm_client),
        }
```
